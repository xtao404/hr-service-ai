# HR AI 平台 — 业务模块全场景测试用例

> 版本：v1.1  
> 适用系统：`hr-admin-ai`（前端）+ `hr-service-ai`（后端）  
> 数据基准：`scripts/mysql/hr_test_data.sql` + `DataInitializer` 种子数据  
> 当前季度：`2026-Q1`  
> **默认配置**：`preset-query.enabled=false`，`text-to-sql.enabled=true`

---

## 一、测试环境与前置条件

| 项目 | 要求 |
|------|------|
| 后端 | `hr-service-ai` 启动于 `http://localhost:8080` |
| 前端 | `hr-admin-ai` 启动于 `http://localhost:5173` |
| 数据库 | MySQL `hr_ai` 库，已执行 `hr_test_data.sql` |
| LLM | Mock 模式或真实 API Key 均可；Mock 下回答为模板化文本，路由与数据逻辑不变 |
| Text-to-SQL | `hr.ai.text-to-sql.enabled=true`（**默认开启**） |
| 预设查询 | `hr.ai.preset-query.enabled=false`（**默认关闭**） |

### 1.1 配置模式与路由对照

| 问题类型 | 默认配置（preset=false） | preset 启用（preset=true） |
|----------|--------------------------|----------------------------|
| 制度/政策 | RAG `KNOWLEDGE` | RAG `KNOWLEDGE` |
| 我的… / 张三假期 | Text-to-SQL | 预设 `PRESET_QUERY` |
| 指定员工（张三的加班） | Text-to-SQL（经理+）/ RAG 降级（员工查他人） | 预设 `NAMED_EMPLOYEE` |
| 部门统计 / 公司概览 | Text-to-SQL | 预设 `DEPT_*` / `COMPANY_OVERVIEW` |
| 复杂分析（对比/排名/超过…） | Text-to-SQL（优先匹配） | Text-to-SQL（优先匹配） |
| 员工查公司统计 | RAG 降级 | RAG 降级 |

> 下文用例标注 **【默认】** 表示 preset 关闭时的预期；标注 **【preset】** 表示需设置 `hr.ai.preset-query.enabled=true` 后执行。

**通用验证点（适用于所有问答类用例）：**

- 页面不展示执行 SQL（问数溯源面板无「执行 SQL」区块）
- 回答末尾含「以上数据来源于HR业务系统」（数据库类查询）
- 流式接口事件顺序：`trace` → `chunk`（多条）→ `done`
- 默认配置下简单数据问句 trace.routeType=`TEXT_TO_SQL`，queryMethod=`text-to-sql`

## 八、模块五：智能问答 — Text-to-SQL（SQL）

路由类型：`TEXT_TO_SQL` | 查询方式：`text-to-sql`

### 8.1 正常业务场景

| 编号 | 角色 | 输入问题 | 预期结果 |
|------|------|----------|----------|
| SQL-01 | MANAGER | 对比各部门加班时长 | 仅返回 D001 技术研发部加班汇总（部门隔离） |
| SQL-02 | HRBP | 对比各部门加班时长 | 返回 D001/D002/D003 三部门对比排名 |
| SQL-03 | HR_ADMIN | 对比各部门加班时长 | 同 SQL-02，全公司部门数据 |
| SQL-04 | MANAGER | 找出满意度低于7分的员工 | D001：赵六(5.8)、钱七(6.2)；不含 D002 褚十三 |
| SQL-05 | HRBP | 找出满意度低于7分的员工 | 含赵六、钱七、褚十三(5.5)、孙八(6.5) 等 |
| SQL-06 | MANAGER | 查询绩效为C且加班超过50小时的员工 | 命中赵六（C / 68.5h） |
| SQL-07 | HRBP | 各部门平均薪酬 | 返回各部门 AVG(base_salary)，HRBP 有薪酬权限 |
| SQL-08 | HRBP | 各部门在职人数 | GROUP BY 部门，D001=8, D002=5, D003=4 |
| SQL-09 | HRBP | 离职风险分布 | 返回 HIGH/CRITICAL 员工列表 |
| SQL-10 | MANAGER | 演示引导步骤 3：对比各部门加班时长 | 触发 Text-to-SQL，展示溯源+图表+行动建议 |

### 8.2 权限与安全拦截

| 编号 | 角色 | 输入/操作 | 预期结果 |
|------|------|-----------|----------|
| SQL-11 | MANAGER | 各部门平均薪酬 | 报错：部门经理无权查询薪酬 / 安全拦截 BIZ_SALARY |
| SQL-12 | EMPLOYEE | 对比各部门加班时长 | 不走 Text-to-SQL（员工+非个人问题被 gate 拦截），降级为 RAG |
| SQL-13 | EMPLOYEE | 我的加班时长排名 | 允许 Text-to-SQL，结果仅含 E001 本人数据 |
| SQL-14 | 任意 | SQL 注入：`'; DROP TABLE biz_employee; --` | 拒绝：含禁止关键字 DROP |
| SQL-15 | 任意 | `SELECT * FROM users` | 拒绝：非白名单表 |
| SQL-16 | 任意 | `SELECT ... UNION SELECT ...` | 拒绝：禁止 UNION |
| SQL-17 | 任意 | 返回超过 100 行 | LIMIT 自动截断为 100 |
| SQL-18 | 任意 | 多条语句 `SELECT 1; SELECT 2` | 拒绝：不允许多条 SQL |
| SQL-19 | EMPLOYEE | LLM 生成无 employee_id 过滤的 SQL | 安全拦截：员工只能查询本人 |
| SQL-20 | MANAGER | LLM 生成无 dept_id 过滤的 SQL | SqlPermissionRewriter 自动注入 `AND e.dept_id = 'D001'` |

### 8.3 展示规范（回归）

| 编号 | 用例名称 | 步骤 | 预期结果 |
|------|----------|------|----------|
| SQL-21 | 页面不展示 SQL | 执行任意 Text-to-SQL 查询 | 聊天正文、溯源面板、参考来源均不出现 SQL 语句 |
| SQL-22 | 溯源信息完整 | 执行 SQL-01 | trace 含 intentLabel=智能 SQL 分析、rowCount、permissionNote，无 generatedSql |
